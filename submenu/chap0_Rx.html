10 Comments pour comprendre les concepts de bases en Base de Données

  1. Comment peut-on définir un système ?

    2. Un système, est un tout constitué d'éléments (Hommes, machines, méthodes, règles...) unis par des rélations. Les élements et les relations sont munis de propriétés. Exemple: une "Entreprise" constitue un système

  2. Comment les éléments interagissent entre eux dans le système ?

    3. Les éléments interagissent entre eux grâce à un processus. Un processus est un ensemble d'opérations exécuté globalement. Lors du processus, des éléments appelés "entrées" sont transformés en d'autres éléments appelés "sorties".
    Exemples:
    1) Pièces détachées ----> Chaine de montage ----> Voitures
    2) Bacheliers ----> Ecole de Formation ----> Diplômes (BTS)

  3. Comment peut-on décrire le système ?

    4. Décrire le système, c'est de déterminer ses éléments, ses relations, son actiivité et son organisation.

  4. Comment peut-on définir un élément ?

    5. Un élément, peut être un objet ou une chose pourvu d'une existence propre.
    Exemples: dans le système "Entreprise", l'élément peut être: "Employés", "Services", "Articles", "Emplacement de stockage",... NB:
    Selon l'auteur, on utilise généralement les termes "Objet"; "Entité" ou "Individu" pour designer un élément.
    L'Entité est munis de propriétés. Une propriété est ce qui caractérise (ou décrit) l'Entité. Exemples de propriétés: "Matricule_Employé", "Nom_Employé", "Reférence" de l'Articles, "designation",...

  5. Comment peut-on définir une relation ?

    6. Une relation est ce qui lit les élements (ou objets/Entités/Individus) entre eux dans un système d'information.
    Exemples: dans le système "Entreprise", La relation peut être:
    1) "est rattaché", qui relie les Entités "Employés" et "Services"
    2) "est stocké", qui relie les Entités "Articles" et "Emplacement de stockage"
    NB:
    Selon l'auteur, on utilise généralement le terme "Association" pour designer une relation
    Comme l'Entité, la relation peut avoir des propriétés. Exemples: "date d'entrée dans le service", "Quantité stockée",...

  6. Comment peut-on définir une cardinalité ?

    7. Une

  7. Comment peut-on définir une occurence ?

    8. Une

  8. Comment peut-on identifier les composants d'un système ?

    9. Une
